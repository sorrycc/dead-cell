# Procedural Audio / SFX (zero-asset WebAudio synth)

## 1. Background

The game ships **no audio at all** — a `grep` for `sound|audio|oscillator|this.sound`
across `src/` finds nothing but unrelated identifiers. Every impact, jump, death, pickup,
and boss entrance is silent. For an action-platformer this is the single biggest
"it feels alive" gap: combat juice (`effects/Effects.ts` — hit-stop, shake, damage
numbers, sparks) lands visually but says nothing.

The project's hard constraint is **programmer-art primitives only — no external sprite or
audio assets** (`README.md`, `package.json` description). Sampled `.wav`/`.mp3` files would
violate it (and `BootScene` deliberately loads nothing — Decision 4). **Procedural WebAudio
synthesis** is the audio equivalent of "colored rectangles": oscillators + noise + gain
envelopes generate every sound at runtime from code, **zero assets**. That is the approach.

Phaser (configured `Phaser.AUTO` in `src/main.ts`) creates one `WebAudioSoundManager` by
default, owns a single shared `AudioContext`, and already wires unlock-on-first-gesture
(the Title→Hub click/keypress resumes a suspended context for free). We reuse that context.

## 2. Requirements Summary

- **Goal:** A small procedural SFX layer that gives combat, movement, economy, set-pieces,
  and menus audible feedback — **no assets**, matching the project's primitives-only rule.
- **Shape:** A `Sound` façade mirroring the role of `effects/Effects.ts` — **one semantic
  call per event** (`sound.jump()`, `sound.hit({damage, crit})`), the synthesis hidden behind
  it (SOLID). Hit sites never touch the audio graph directly.
- **Integration:** Inject into `Player` like `hitboxPool`/`projectilePool` (optional,
  null-safe); call directly in `GameScene` at the existing `effects.hit()` sites; add menu
  blips to `HubScene`/`TitleScene`.
- **Non-goals (YAGNI this slice):** background music tracks, a mixer UI, positional/3-D
  audio, per-biome ambience. A global **mute toggle (M)** is included (cheap + expected).

## 3. Acceptance Criteria

1. New `src/audio/Sound.ts` — a Phaser-coupled synth façade. **No `load.*` calls, no asset
   files.** Reuses Phaser's shared `AudioContext` (`scene.sound.context`).
2. **Combat:** player melee hit + projectile hit (volume/brightness scaled by damage; a
   distinct **backstab/crit** timbre), player hurt, enemy death.
3. **Movement:** jump, dodge, land, melee swing (timbre varies by weapon type), bow shot.
4. **Economy/feedback:** pickup (timbre per kind: cell/gold/scroll/weapon/heal), flask drink,
   weapon swap.
5. **Set-pieces:** boss/miniboss spawn, boss/miniboss defeat (a short flourish), level
   transition, player death.
6. **Menus:** Hub cursor move + buy/confirm; Title start.
7. Respects Phaser global mute/volume (`scene.sound.mute`/`.volume`); an **M** key toggles
   mute. Degrades to a **silent no-op** when WebAudio is unavailable (headless/NoAudio) — never
   throws.
8. **Identity / purity:** all pure config modules and `scripts/verify-gen.mjs` are
   **unchanged** (`Sound` is imported only by Phaser scenes/entities). `npm run verify` stays
   green; `npm run typecheck` passes.
9. No steady-state allocation churn beyond the short-lived `OscillatorNode`/`GainNode` per
   sound, which auto-disconnect on `stop` and GC. A throttle prevents transient pile-ups
   (e.g. a spitter fan landing 3 hits in one frame).

## 4. Problem Analysis

- **Approach A — ship sampled audio files + `this.load.audio`** → rejected: violates the
  no-external-assets constraint; adds a load phase to the currently empty `BootScene`; bloats
  the repo/bundle.
- **Approach B — feed generated `AudioBuffer`s into Phaser's `this.sound.add().play()`** →
  rejected: Phaser's sound API is built around decoded buffers/markers; routing synthesized
  one-shots through it is more plumbing than calling the raw context, with less envelope
  control. We only want fire-and-forget transients.
- **Approach C (chosen) — a thin WebAudio synth façade** mirroring `effects/Effects.ts`:
  grab the `AudioContext` from Phaser's `WebAudioSoundManager`, synthesize each SFX from
  `OscillatorNode`/noise + a `GainNode` envelope. Zero assets, full control, integrates with
  Phaser's mute/volume + unlock-on-gesture. Same "one call per event" discipline the visual
  juice façade already uses.

## 5. Decision Log

**1. Where does the `AudioContext` come from?**
- Options: A) `new AudioContext()` we own; B) reuse Phaser's `scene.sound.context`.
- Decision: **B.** Phaser owns ONE shared `WebAudioSoundManager` (`game.sound`) across all
  scenes, so every `Sound` instance shares one context. Phaser's existing unlock listeners
  resume it on the first user gesture (the Title requires a click/key to advance, so the
  context is running before any gameplay sound). `scene.sound.mute`/`.volume` integrate for
  free. **Fallback:** if the manager is `NoAudioSoundManager` (or lacks `.context`), store
  `ctx = null` and every method no-ops (AC7) — headless/`verify` safe.

**2. How is `Sound` wired to the entities (coupling)?**
- Options: A) emit events on `scene.events`, a listener plays; B) inject the façade and call
  it directly (the `Effects` pattern).
- Decision: **B.** `Player` already holds `scene`/`hitboxPool`/`projectilePool` refs and
  `GameScene` already calls `effects.hit()` directly at every impact — adding `sound` is
  consistent and avoids a second event bus (KISS). `Player` gets an **optional** `sound` ctor
  param (default `null`); a null sound is a no-op, so a Phase-1/headless `Player` is
  byte-identical (the additive identity).

**3. Synthesis primitives (keep it tiny).**
- Decision: two private helpers — `_tone({freq, type, dur, gain, sweepTo})` (an
  `OscillatorNode` → `GainNode` with a short attack + exponential decay, optional linear
  frequency sweep) and `_noise({dur, gain, type, freq})` (a small white-noise buffer through
  a `BiquadFilter`). Every semantic method composes these. YAGNI on a general synth graph.

**4. Master gain + soft limiting.**
- Decision: one master `GainNode` per instance → `destination`, base ~`0.32 × sound.volume`,
  skipped entirely when `sound.mute`. Keeps stacked transients from clipping when several
  hits land the same frame.

**5. Hit-stop boundary.**
- Decision: SFX schedule on the **WebAudio clock** (`ctx.currentTime`), NOT the gameplay dt.
  The impact "pop" should be audible *during* the micro-freeze — identical to how
  `Effects.tick` runs sparks/numbers on REAL dt (Decision 24/26). No coupling to
  `hitstopTimer`.

**6. Transient throttle.**
- Decision: a per-key min-interval (`~30ms`) using `ctx.currentTime` — a multi-projectile
  frame fires one hit transient, not eight. Cosmetic + off the seeded RNG, so determinism /
  the level pin are unaffected (audio is non-deterministic churn like particles, which
  already use `Math.random`).

**7. Mute toggle key.**
- Decision: bind **M** in `core/Input.ts` (edge-detected, sole-owned like the other keys) →
  GameScene toggles `this.sound.mute` (which flips Phaser's global `game.sound.mute`). One
  new `InputSnapshot` field `mutePressed`. Small, expected, and aids manual verification.

## 6. Design

### 6.1 `src/audio/Sound.ts` (new)

```ts
export class Sound {
  private sm: Phaser.Sound.BaseSoundManager
  private ctx: AudioContext | null
  private master: GainNode | null
  private _last: Record<string, number> // per-key throttle stamps (ctx time).

  constructor(scene: Phaser.Scene) {
    this.sm = scene.sound
    const ctx = (this.sm as any).context ?? null      // WebAudio only; null on NoAudio.
    this.ctx = ctx && typeof ctx.createOscillator === 'function' ? ctx : null
    this.master = this.ctx ? this.ctx.createGain() : null
    if (this.master && this.ctx) this.master.connect(this.ctx.destination)
    this._last = {}
  }

  // ── primitives ──
  private _gateOk(key: string, minGap = 0.03): boolean { /* throttle + mute + ctx guard */ }
  private _tone(o): void { /* osc → gain env → master */ }
  private _noise(o): void { /* noise buffer → biquad → gain env → master */ }

  // ── semantic SFX (one per event) ──
  swing(weaponType: string) {}      // melee whoosh; hammer lower / spear higher.
  shoot() {}                        // bow twang (downward sweep).
  hit({ damage, crit }) {}          // noise thud + tone; crit = brighter + a ring.
  hurt() {}                         // harsh descending square blip.
  enemyDie() {}                     // short downward noise+tone.
  jump() {}  doubleJump() {}  dodge() {}  land() {}   // movement (doc 3 calls these too).
  pickup(kind) {}                   // ascending blip; pitch per kind.
  flask() {}  swap() {}
  bossSpawn() {}  bossDefeat() {}  transition() {}  death() {}
  uiMove() {}  uiSelect() {}
  get mute() {} set mute(v) {}      // proxies game.sound.mute.
}
```

Every method early-returns when `this.ctx` is null or muted, so the whole class is a safe
no-op without WebAudio.

### 6.2 `Player` wiring (`src/entities/Player.ts`)

- Ctor gains an optional 6th param `sound: Sound | null = null`, stored as `this.sound`.
- Call sites (all `this.sound?.x()` — null-safe):
  - jump launch (step 3, ~`Player.ts:435`) → `jump()`
  - dodge start (~`:385`) → `dodge()`
  - land squash edge (step 6, ~`:466`) → `land()`
  - `_startSwing` melee branch → `swing(weapon.type)`; ranged branch → `shoot()`
  - `onHit` (player took damage) → `hurt()`

### 6.3 `GameScene` wiring (`src/scenes/GameScene.ts`)

- In `create()` (next to `this.effects = new Effects(...)`, ~`:239`):
  `this.sound = new Sound(this)`; pass it as the new `Player` arg (~`:259`).
- Call sites beside the existing `effects.hit()` / camera-flash points:
  - `_onPlayerHitEnemy` (~`:1353`) → `sound.hit({ damage: result.damage, crit: result.isBackstab })`
  - `_onProjectileHitEnemy` (~`:1395`) → same
  - `enemy.onDeath` hook (~`:1172`) → `sound.enemyDie()`
  - `_collectPickup` (~`:1458`) → `sound.pickup(kind)`
  - `_drinkFlask` (~`:1493`) → `sound.flask()`
  - `_swapWeapon` (on a real swap, ~`:1037`) → `sound.swap()`
  - boss/miniboss spawn (`_spawnBoss`/`_spawnMiniboss`) → `sound.bossSpawn()`
  - `_onBossDefeated` / `_completeRun` → `sound.bossDefeat()` / flourish
  - level change (`_onDoorOverlap`/`_nextLevel`, ~`:787`) → `sound.transition()`
  - `_onPlayerDeath` (~`:1650`) → `sound.death()`
  - in `update()` input handling: `if (inputState.mutePressed) this.sound.mute = !this.sound.mute`

### 6.4 Menus

`HubScene` and `TitleScene` each construct their own `new Sound(this)` (shares the one
context). Hub cursor move → `uiMove()`, buy/START → `uiSelect()`; Title start → `uiSelect()`.

### 6.5 Input (`src/core/Input.ts`)

Add `m: KC.M` to the key map; `mutePressed = JustDown(keys.m)`; add `mutePressed` to
`InputSnapshot`. (No other key changes.)

### Identity safety

No pure module (`config/*`, `combat/*` pure parts, `core/RunState`, `core/MetaState`,
`world/LevelGenerator`) is touched. `scripts/verify-gen.mjs` imports none of these new
symbols. A `Player` constructed without a `sound` (or in a NoAudio env) behaves exactly as
today.
